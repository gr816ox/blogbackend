# 认证模块后端实现清单

> 目标：只有主账户（ADMIN）能新增/编辑/删除；其他用户只能浏览。

## 1. 数据与模型
- users 表（最小可用）：`id`, `username`, `password_hash`, `enabled`, `role`
- `role` 取值：`ADMIN` / `USER`

## 2. 接口设计
### 2.1 登录
- `POST /api/auth/login`
  - 入参：`{ username, password }`
  - 返回：`{ token, user: { id, username, role } }`

### 2.2 当前用户
- `GET /api/auth/me`
  - Header：`Authorization: Bearer <token>`
  - 返回：当前用户信息（用于前端刷新后恢复登录态）

## 3. 权限控制（核心）
### 3.1 允许匿名访问
- `GET /api/articles`
- `GET /api/articles/{id}`

### 3.2 仅 ADMIN 允许
- `POST /api/articles`
- `PUT /api/articles/{id}`
- `DELETE /api/articles/{id}`

## 4. Spring Security + JWT 方案（推荐）
- 配置 `SecurityFilterChain` 规则：
  - `/api/auth/**`：`permitAll`
  - `GET /api/articles/**`：`permitAll`
  - `POST/PUT/DELETE /api/articles/**`：`hasRole("ADMIN")`
  - 其他：`authenticated`
- JWT Filter：解析 `Authorization`，构建 `Authentication`
- 可选：Controller 上使用 `@PreAuthorize("hasRole('ADMIN')")`

### 4.1 最小可用示例（Spring Security 6）
```java
@Configuration
@EnableMethodSecurity
public class SecurityConfig {

  @Bean
  public SecurityFilterChain securityFilterChain(HttpSecurity http) throws Exception {
    http
      .csrf(csrf -> csrf.disable())
      .sessionManagement(sm -> sm.sessionCreationPolicy(SessionCreationPolicy.STATELESS))
      .authorizeHttpRequests(auth -> auth
        .requestMatchers("/api/auth/**").permitAll()
        .requestMatchers(HttpMethod.GET, "/api/articles/**").permitAll()
        .requestMatchers(HttpMethod.POST, "/api/articles/**").hasRole("ADMIN")
        .requestMatchers(HttpMethod.PUT, "/api/articles/**").hasRole("ADMIN")
        .requestMatchers(HttpMethod.DELETE, "/api/articles/**").hasRole("ADMIN")
        .anyRequest().authenticated()
      );

    // TODO: add JWT filter before UsernamePasswordAuthenticationFilter
    // http.addFilterBefore(jwtFilter, UsernamePasswordAuthenticationFilter.class);

    return http.build();
  }
}
```

## 5. DELETE 415 避坑（必须）
- 正确写法：`DELETE /api/articles/{id}`
- 参数：`@PathVariable Long id`
- 不要 `@RequestBody`，不要设置 `consumes = application/json`

## 6. 落地检查清单
- 写接口严格区分读/写权限
- 写操作必须校验 `ADMIN` 角色，非 ADMIN 返回 403
- 登录成功返回 `token + role` 供前端使用
